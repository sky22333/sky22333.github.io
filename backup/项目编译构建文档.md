> ## npm包管理器国内镜像

### 当前终端有效
```
# Linux/macOS
export npm_config_registry=https://registry.npmmirror.com

# Windows PowerShell
$env:npm_config_registry="https://registry.npmmirror.com"

# Windows CMD
set npm_config_registry=https://registry.npmmirror.com
```

### 永久配置
```
npm config set registry https://registry.npmmirror.com
```
恢复默认
```
npm config set registry https://registry.npmjs.org
```

```
│  ├─ Properties/
│  │  └─ launchSettings.json  # 调试配置（VS / Rider）
│  ├─ appsettings.json     # 配置文件（可选）
│  ├─ Models/              # 存放数据模型类
│  │  └─ User.cs
│  ├─ Services/            # 存放服务类
│  │  └─ EmailService.cs
│  ├─ Controllers/         # MVC 或 API 项目才有
│  │  └─ UserController.cs
│  └─ wwwroot/             # Web 静态资源（Web 项目才有）
├─ .gitignore              # git 忽略文件
├─ global.json             # SDK 版本锁定（可选）
└─ README.md
```

常用命令

```
# 恢复项目依赖，下载 NuGet 包并生成依赖缓存
dotnet restore

# 安装代码格式化
dotnet tool install -g dotnet-format

# 执行代码格式化
dotnet format

# 编译项目并执行静态代码分析
dotnet build -warnaserror -p:AnalysisLevel=latest

# 运行单元测试，确保代码逻辑正确
dotnet test

# 热重载运行
dotnet watch run

# 生成可部署的生产版本，放在 publish 目录
```

```
    fn MessageBoxW(hWnd: isize, lpText: *const u16, lpCaption: *const u16, uType: u32) -> i32;

fn main() {
    unsafe {
        MessageBoxW(
            0,
            "环境正常！\0".encode_utf16().collect::<Vec<u16>>().as_ptr(),
            "验证成功\0".encode_utf16().collect::<Vec<u16>>().as_ptr(),

        );

}
```

### 使用 uniffi 将 Rust 项目打包为 Android AAR

#### 核心思路
接口声明 → 工具生成绑定 → 标准 NDK 编译 → 打包，完全免去手写 JNI 胶水代码。

#### 具体步骤

1. **Rust 侧暴露接口**
   - 用 `#[uniffi::export]` 宏标注要公开的函数/类型
   - 或编写一个 UDL 文件描述接口
   - 在 `Cargo.toml` 中设置 `crate-type = ["cdylib"]` 并添加 `uniffi` 依赖

2. **生成 Kotlin 绑定**
   - 用 `uniffi-bindgen` 命令行工具
   - 根据上一步的接口定义自动生成 Kotlin 调用代码

3. **编译原生 .so**
   - 使用 `cargo-ndk` 编译出 `arm64-v8a` 等架构的 `.so` 动态库

4. **在 Android 项目中集成**
   - 将生成的 Kotlin 文件和 `.so` 文件放入 Android 项目目录
   - 或通过 Gradle 插件实现自动化

---

### Rust 项目封装为 Android AAR

#### 前提
原项目有 `lib.rs` 且公开了必要 API（`pub`）。

#### 步骤

1. **新建包装项目**，在 `Cargo.toml` 中依赖原项目 + `uniffi`
2. **写 `lib.rs`**，用 `#[uniffi::export]` 暴露 `start()` / `stop()` 等入口
3. **运行 `uniffi-bindgen`** 生成 Kotlin 绑定代码
4. **运行 `cargo-ndk`** 编译出 `arm64-v8a` 等 `.so` 文件
5. **放入 Android 项目** `jniLibs/` 目录，Gradle 构建即出 AAR

#### 关键
- 原项目代码**零修改**
- 包装项目**只写接口转发**，不写业务
- 和 Go `gomobile bind` 思路完全一致
