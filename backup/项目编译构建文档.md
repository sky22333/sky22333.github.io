## 主流编程语言的安装依赖和构建运行命令

> [!TIP]
> ## Java (Spring Boot) 💻

#### 基本代码结构
```
my-springboot-project/
├── src/
│ ├── main/
│ │ ├── java/
│ │ │ └── com/
│ │ │ └── example/
│ │ │ └── MySpringBootApplication.java
│ │ └── resources/
│ │ ├── application.properties
│ │ └── static/
│ └── test/
│ └── java/
│ └── com/
│ └── example/
│ └── MySpringBootApplicationTests.java
├── .gitignore
├── build.gradle
└── pom.xml
```

#### 安装依赖和构建

#### Maven
```
mvn clean package
```
#### Gradle
```
./gradlew build
```
#### 运行构建出来的`jar`包（一般在项目的`target/you.jar`目录中）
```
java -jar target/you.jar
```

> [!TIP]
> ## Go 💻


#### 基本代码结构
```
my-go-project/
├── main.go
└── README.md
```

#### 清理缓存
```
go clean -cache
```
#### 初始化模块
```
go mod init myapp
```
#### 自动识别并安装依赖
```
go mod tidy
```
#### 在线运行
```
go run main.go
```

#### 质量检查
```
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest
golangci-lint run
```
```
go install honnef.co/go/tools/cmd/staticcheck@latest
staticcheck ./...
```
#### 格式化代码
```
go fmt ./...
```

#### 将所有依赖和静态链接一起打包，并优化包大小
```
GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -ldflags="-s -w" -o myapp
```

#### Windows系统构建
```
$env:GOOS="windows"; $env:GOARCH="amd64"; $env:CGO_ENABLED="0"; go build -ldflags="-s -w" -o myapp.exe
```
```
$env:GOOS="linux"; $env:GOARCH="amd64"; $env:CGO_ENABLED="0"; go build -ldflags="-s -w" -o myapp-amd64
```

## PHP 💻

基本代码机构
```
your-project/
├── index.php
├── config/
│   └── config.php
├── src/
│   ├── Controller/
│   ├── Model/
│   ├── View/
│   └── ...
├── public/
│   ├── css/
│   ├── js/
│   └── images/
├── vendor/
├── .env
├── .gitignore
├── composer.json
├── composer.lock
└── README.md
```

#### 安装依赖

如果项目使用`Composer`进行依赖管理，请在项目目录中运行`composer install`
```
cd /var/www/html/your-project
composer install
```

### 运行

安装`PHP`和`PHP-FPM` 并配置配置`nginx`，确保你的`public/index.php`是项目的入口文件，`nginx`运行目录一般都是这个目录。

> ## C#

环境安装：https://dotnet.microsoft.com/zh-cn/download/dotnet

基本代码结构
```
MyApp/
├─ MyApp.sln               # 解决方案文件（可选，多项目时使用）
├─ MyApp/                  # 项目目录
│  ├─ MyApp.csproj         # 项目文件（包含依赖、编译选项）
│  ├─ Program.cs           # 主程序入口
│  ├─ Properties/
│  │  └─ launchSettings.json  # 调试配置（VS / Rider）
│  ├─ appsettings.json     # 配置文件（可选）
│  ├─ Models/              # 存放数据模型类
│  │  └─ User.cs
│  ├─ Services/            # 存放服务类
│  │  └─ EmailService.cs
│  ├─ Controllers/         # MVC 或 API 项目才有
│  │  └─ UserController.cs
│  └─ wwwroot/             # Web 静态资源（Web 项目才有）
├─ .gitignore              # git 忽略文件
├─ global.json             # SDK 版本锁定（可选）
└─ README.md
```

常用命令

```
# 恢复项目依赖，下载 NuGet 包并生成依赖缓存
dotnet restore

# 安装代码格式化
dotnet tool install -g dotnet-format

# 执行代码格式化
dotnet format

# 编译项目并执行静态代码分析
dotnet build -warnaserror -p:AnalysisLevel=latest

# 运行单元测试，确保代码逻辑正确
dotnet test

# 热重载运行
dotnet watch run

# 生成可部署的生产版本，放在 publish 目录
dotnet publish -c Release
```

### 创建一个demo
```
cargo new demo
cd demo
```

把 `src\main.rs` 里的内容全部替换成下面这段代码
```
// 告诉编译器要链接 Windows 的 user32 库
#[link(name = "user32")]
extern "system" {
    // 声明一个 Windows API 函数：消息框
    fn MessageBoxW(hWnd: isize, lpText: *const u16, lpCaption: *const u16, uType: u32) -> i32;
}

fn main() {
    // 调用系统的 MessageBox 弹窗
    unsafe {
        MessageBoxW(
            0,
            "环境完美！\0".encode_utf16().collect::<Vec<u16>>().as_ptr(),
            "验证成功\0".encode_utf16().collect::<Vec<u16>>().as_ptr(),
            0x40, // 信息图标
        );
    }
    println!("控制台也能输出！");
}
```
编译
```
cargo build
```

#### 常用开发命令

| 分类 | 命令 | 作用 |
|:---|:---|:---|
| 项目管理 | `cargo new <项目名>` | 创建新的二进制项目 |
| 项目管理 | `cargo new --lib <库名>` | 创建新的库项目 |
| 项目管理 | `cargo init` | 在当前目录初始化一个 Rust 项目 |
| 编译与运行 | `cargo build` | 编译项目（debug 模式，未优化） |
| 编译与运行 | `cargo build --release` | 编译项目（release 模式，高度优化） |
| 编译与运行 | `cargo run` | 编译并运行（debug 模式） |
| 编译与运行 | `cargo run --release` | 编译并运行（release 模式） |
| 编译与运行 | `cargo check` | 只检查代码能否通过编译，不生成 exe，速度极快 |
| 清理依赖 | `cargo clean` | 删除项目中的 target 文件夹 |
| 代码质量 | `cargo fmt` | 自动格式化代码风格 |
| 代码质量 | `cargo clippy` | 运行官方代码检查工具，给出改进建议 |
| 代码质量 | `cargo test` | 运行项目中所有单元测试和集成测试 |
| 代码质量 | `cargo test -- --nocapture` | 运行测试的同时显示 println! 输出 |
| 代码质量 | `cargo bench` | 运行性能基准测试（需要 nightly 工具链） |
| 依赖管理 | `cargo add <库名>` | 添加依赖到 Cargo.toml |
| 依赖管理 | `cargo add <库名> --features <特性>` | 添加依赖并启用特定 features |
| 依赖管理 | `cargo remove <库名>` | 移除依赖 |
| 依赖管理 | `cargo update` | 更新依赖到 Cargo.lock 允许的最新兼容版本 |
| 依赖管理 | `cargo tree` | 以树状图显示整个依赖关系 |
